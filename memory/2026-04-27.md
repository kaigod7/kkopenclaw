
## 晚間天氣卡降雨問題

**問題**：`evening_news.sh` 的 `build_evening_card` 逐小時降水概率數據來自 `hourly.precipitation`，但當降級到 QWeather 或 Open-Meteo 時，返回的 `probability` 字段格式可能與 Caiyun 不同（如 Caiyun 返回 0-1 小數，QWeather 返回 0-100 整數），導致 `int(h["datetime"][11:13])` 解析出錯或 `rain_row` 過濾條件不匹配。

**根本原因**：天氣 API 降級後數據格式不一致，build_evening_card 沒有針對不同數據源做適配。

**教訓**：
- 天氣數據降級不能只保證「能拿到數據」，還要保證數據格式一致
- 所有天氣腳本統一用 `weather_lib.fetch_weather()`，數據轉換由 lib 負責，卡片的構建邏輯只認標準格式
- 天氣腳本改完後要在**不同時段**都跑一遍測試（因為不同 API 在不同時間回應不同）
